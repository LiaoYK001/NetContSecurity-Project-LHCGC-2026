# 成员 B 图像与多模态融合指南

适用角色：图像模型与多模态融合负责人  
目标周期：第 1 天到第 7 天  
核心目标：跑通图像分支，设计可落地的多模态融合方案，并提供模型结构、融合结果和消融材料。

## 0. 先看这一段：白话解释

你不需要一开始就懂计算机视觉。成员 B 的工作可以理解成两件事：第一，让电脑把图片变成一串数字特征；第二，把 A 给的文本特征、图片特征和行为特征合在一起，让 C 能统一评估。

所谓 ResNet，就是一个网上已经训练好的图片识别模型。我们不一定要重新训练它，可以直接拿它当“图片特征提取器”。所谓多模态融合，就是把不同来源的信息放在一起判断，例如文本说像诈骗、图片也像广告、账号行为也异常，那么风险更高。

这门课不要求你做复杂视觉研究。保底目标是：图片能读取、能用预训练 ResNet 提取特征、能把图像分支作为多模态方案的一部分讲清楚。

## 0.1 保底方案 / 标准方案 / 拔高方案

| 方案 | 你要做什么 | 做到什么程度算合格 |
| --- | --- | --- |
| 保底方案 | 不训练图像模型，只用预训练 ResNet18 提取图片特征。 | 能说明图片如何输入、如何变成向量、如何交给 C。 |
| 标准方案 | 使用 ResNet 图像特征 + A 的文本向量 + 行为特征，做拼接后分类。 | 能跑出融合模型结果，并和文本单模态做对比。 |
| 拔高方案 | 增加 OCR 或注意力融合。 | OCR 能提取图片中文字，或注意力融合能作为扩展实验说明。 |

## 0.2 手把手步骤

1. 先问 A：数据表里的 `image_path` 是否有值，图片文件是否真实存在。
2. 先不要训练模型，先写清楚图片处理流程：读取图片、转成 RGB、缩放到 `224 x 224`、归一化。
3. 使用 torchvision 里的预训练 ResNet18 或 ResNet50，当成现成基础模型。
4. 如果图片少或质量差，不要强行训练，把图片分支定位为辅助特征。
5. 第 3-4 天先输出图像向量或图像预测概率。
6. 第 5 天把 A 的文本向量、图像向量、行为特征拼接。
7. 如果拼接训练很麻烦，就退回分数加权：文本分数 0.5，图像分数 0.3，行为分数 0.2。
8. 把模型结构图交给 E，把融合说明交给 D，把预测结果交给 C。

## 0.3 现成模型和工具怎么选

| 场景 | 推荐选择 | 原因 |
| --- | --- | --- |
| 只想稳稳跑通 | ResNet18 预训练模型 | 小、快、资料多。 |
| 图片数量较多 | ResNet50 预训练模型 | 特征能力更强，但稍慢。 |
| 图片是广告海报或截图 | OCR 作为可选补充 | 图片里的文字往往是关键信号。 |
| 没有 GPU | 冻结 ResNet，只提取特征 | 不训练主体，普通电脑也更容易跑。 |
| 融合模型难写 | 分数加权融合 | 不需要复杂神经网络，也能体现多模态思想。 |

## 0.4 失败时怎么办

| 问题 | 保底处理 |
| --- | --- |
| 图片很少 | 把图像分支写成辅助实验，主结果依靠文本和行为。 |
| 图片加载失败 | 记录失败比例，缺失图片用零向量或跳过。 |
| ResNet 效果不好 | 强调公开数据图片信号弱，用消融实验说明原因。 |
| OCR 装不上或太慢 | 放弃 OCR，把它写成未来改进。 |
| 融合后不如文本模型 | 如实保留结果，说明多模态不是必然提升，取决于数据质量。 |

## 0.5 每天做到什么程度算合格

| 时间 | 合格线 |
| --- | --- |
| 第 1 天 | 图片字段和缺失图片处理方式已确定。 |
| 第 2 天 | 至少能成功读取几张图片并统一尺寸。 |
| 第 3-4 天 | ResNet 能输出图像向量或图像预测结果。 |
| 第 5 天 | 能完成一次文本、图像、行为的融合尝试。 |
| 第 6 天 | 有消融实验说明，知道图像分支贡献多少。 |
| 第 7 天 | 结构图、融合说明和预测结果都交出去。 |

## 0.6 可以直接写进报告/PPT 的说法

本项目的图像分支采用预训练 ResNet 提取图片特征，避免在短周期内从零训练图像模型。图像特征与文本语义特征、用户行为特征进行拼接后输入分类器，用于验证多模态信息是否能补充单纯文本检测的不足。如果图像分支提升不明显，则通过消融实验说明当前数据中图片信号有限，并将 OCR 和更复杂融合方法作为后续改进方向。

## 1. 你最终要交付什么

| 交付物 | 最晚时间 | 验收标准 |
| --- | --- | --- |
| 图片输入规范 | 第 1 天 | 明确图片路径、格式、尺寸、缺失图片处理方式。 |
| 图像基线结果 | 第 3-4 天 | ResNet18 或 ResNet50 能独立输出预测结果或图像向量。 |
| OCR 可选方案说明 | 第 4 天 | 说明是否启用 OCR，不启用也要写原因。 |
| 多模态融合第一版 | 第 5 天 | 能把文本向量、图像向量、行为特征拼接并输出分类结果。 |
| 消融实验材料 | 第 6 天 | 至少支持文本、图像、行为、融合模型的对比。 |
| 第 7 天交接材料 | 第 7 天 | 模型结构图、融合方式说明、指标表、300-500 字技术解释。 |

## 2. 第 1 天：确定图像输入规范

### 操作步骤

1. 和 A 确认 `image_path` 字段如何填写。
2. 约定没有图片的样本如何处理：
   - 方案一：丢弃没有图片的样本，只做纯多模态子集。
   - 方案二：保留样本，图像向量用全 0 填充。
   - 推荐先用方案二，样本利用率更高。
3. 确认图片格式：优先支持 `jpg`、`jpeg`、`png`。
4. 统一图片尺寸：ResNet 常用 `224 x 224`。
5. 和 C 确认图像向量保存格式，例如 `sample_id + image_embedding`。

### 当天检查清单

- 已明确缺失图片处理策略。
- 已和 A 对齐图片路径字段。
- 已和 C 对齐图像向量输出格式。
- 已告诉 D 图像分支在系统架构中的位置。

## 3. 第 2 天：准备图片预处理流程

### 推荐流程

```text
读取图片路径
-> 检查文件是否存在
-> 转成 RGB
-> resize 到 224 x 224
-> normalize
-> 输入 ResNet
```

### 需要处理的异常

| 异常 | 处理方式 |
| --- | --- |
| 图片路径为空 | 使用零向量或标记缺失。 |
| 图片文件损坏 | 跳过并记录 `sample_id`。 |
| 图片尺寸过小 | resize 后保留，但记录风险。 |
| 图片中有二维码或隐私 | 不提交原图，只给脱敏说明。 |

## 4. 第 3-4 天：跑通图像基线模型

### 模型选择

| 情况 | 推荐模型 |
| --- | --- |
| 时间紧、电脑一般 | ResNet18 预训练特征提取。 |
| 有较多图片样本 | ResNet50 特征提取或微调分类头。 |
| 图片主要是截图文字 | 加 OCR，把识别出的文字交给 A 的文本分支。 |

### 推荐做法

1. 先冻结 ResNet 主体，只提取图像向量。
2. 在图像向量后接一个简单分类器。
3. 输出图像单模态指标。
4. 保存图像向量，供第 5 天融合使用。

### 当天检查清单

- 图像模型能处理至少一批样本。
- 图像向量维度已记录。
- 图像单模态指标已交给 C。
- 图片预处理流程已写成文字，交给 D。

## 5. OCR 可选路线

OCR 不是必做项，只有在图片中明显有广告字、截图字、二维码旁边文字时才做。

### 使用原则

| 情况 | 是否建议 OCR |
| --- | --- |
| 大量图片是纯照片 | 不建议，收益可能不高。 |
| 大量图片是广告海报 | 建议，文字信号很强。 |
| 时间不足 | 不建议，避免阻塞主线融合。 |
| OCR 环境安装失败 | 放弃 OCR，把它写成后续改进。 |

OCR 结果不要直接作为最终结论，应该作为额外文本字段交给 A，再进入文本模型。

## 6. 第 5 天：多模态融合第一版

### 输入

| 输入 | 来源 |
| --- | --- |
| 文本向量或文本预测概率 | 成员 A |
| 图像向量或图像预测概率 | 成员 B |
| 用户行为特征 | 成员 A |
| 标签和数据划分 | 成员 A/C |

### 推荐融合方式

优先做特征拼接：

```text
文本向量 + 图像向量 + 行为特征
-> 拼接成一个总向量
-> MLP 分类器
-> 输出风险概率
```

如果时间不够，使用分数加权：

```text
文本模型概率 * 0.5 + 图像模型概率 * 0.3 + 行为模型概率 * 0.2
```

### 当天验收标准

- 融合模型能跑完训练或推理。
- C 能拿到融合模型预测结果。
- D 能拿到一段清楚的融合方法说明。
- E 能拿到一张简洁的模型结构图。

## 7. 第 6 天：消融实验

需要配合 C 完成以下对比：

| 实验 | 目的 |
| --- | --- |
| 仅文本 | 验证文本内容是否有效。 |
| 仅图像 | 验证图片是否提供额外信息。 |
| 仅行为 | 验证账号异常特征是否有效。 |
| 文本 + 图像 | 验证双模态提升。 |
| 文本 + 图像 + 行为 | 最终方案。 |

你需要解释：融合模型提升来自哪里，失败样例可能是什么原因。

## 8. 第 7 天：交接包

### 交给 C

- 图像模型预测结果。
- 图像向量维度和保存格式。
- 融合模型预测结果。
- 消融实验所需的各组输入。

### 交给 D

- 图像预处理流程。
- ResNet 选择原因。
- 融合方式说明。
- 模型结构图说明。
- 300-500 字技术解释。

### 交给 E

- 一张模型结构图。
- 一张消融实验柱状图建议。
- 2-3 个图片误判案例解释。

## 9. 常见问题和补救方案

| 问题 | 补救方案 |
| --- | --- |
| 图片太少 | 保留图像分支作为辅助实验，最终强调文本和行为特征。 |
| 图片加载失败很多 | 记录失败比例，先用可读取样本跑通模型。 |
| 融合后效果没有提升 | 做消融分析，说明图像信号弱或样本分布不匹配。 |
| OCR 太慢或装不上 | 不阻塞主线，把 OCR 写入未来改进。 |
| 向量维度对不上 | 先打印每个输入维度，再统一拼接前的数组形状。 |

## 10. 每天结束前自查

- 图像输入是否能和 A 的数据表对齐。
- 图像向量是否能被 C 读取。
- 融合方法是否足够简单、可解释、可复现。
- 是否给 D/E 提供了能写报告和做 PPT 的图表或说明。
- 是否避免提交原始图片和可能含隐私的截图。
