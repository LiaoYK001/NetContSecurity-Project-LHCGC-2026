# 成员 A 数据与文本模型指南

适用角色：数据负责人兼文本模型负责人  
目标周期：第 1 天到第 7 天  
核心目标：把“可训练、可复现、可解释”的文本与行为数据准备好，并跑通文本基线模型和 BERT/中文 RoBERTa 文本分支。

## 0. 先看这一段：白话解释

你不需要一开始就懂 NLP。成员 A 的工作可以理解成三件事：第一，把“能不能训练”的数据整理出来；第二，先用最简单的文本分类方法跑一个结果；第三，如果时间允许，再用网上现成的中文预训练模型做一个更像深度学习项目的版本。

所谓“文本模型”，就是让电脑根据一句话或一段评论判断它更像正常内容还是风险内容。所谓“训练集”和“测试集”，就是一部分数据给模型学习，另一部分数据拿来考试。所谓“标签”，就是每条样本的标准答案，例如 `normal` 表示正常，`risk` 表示风险。

这门课是选修课探索性项目，不是要求你做出论文级模型。保底目标是：数据来源说得清楚、字段整理规范、能跑出一个文本分类结果、能解释为什么效果好或不好。

## 0.1 保底方案 / 标准方案 / 拔高方案

| 方案 | 你要做什么 | 做到什么程度算合格 |
| --- | --- | --- |
| 保底方案 | 找一个公开文本数据集，整理成 CSV，用 TF-IDF + 逻辑回归做二分类。 | 能输出准确率、精确率、召回率、F1 值，并给出 3 条误判样例。 |
| 标准方案 | 在保底方案基础上，使用 `bert-base-chinese` 或中文 RoBERTa 提取文本向量或训练分类头。 | 能说明 BERT 比 TF-IDF 多用了上下文语义信息，结果能交给 C 评估。 |
| 拔高方案 | 增加用户行为特征，例如链接数量、重复比例、发帖频率、互动比。 | 能证明“文本 + 行为”比只看文本更完整，即使提升不明显也能解释原因。 |

## 0.2 手把手步骤

1. 打开仓库，先运行 `uv sync`，确认环境能安装。
2. 新建一张数据字段表，先不用写代码，先把列名定下来：`sample_id`、`text`、`image_path`、`behavior_features`、`label`、`source`、`split`。
3. 去公开数据平台搜索关键词，例如 `Chinese spam text dataset`、`social media spam dataset`、`fraud text dataset`。
4. 找到数据后先看 20 条样本，判断它是否真的和垃圾广告、诈骗、有害内容有关。
5. 把标签先简化成二分类：正常写 `normal`，风险写 `risk`。
6. 用表格或脚本清洗文本，先处理空文本、重复文本、手机号、邮箱、链接和账号名。
7. 第 3 天先跑 TF-IDF + 逻辑回归，不要直接卡在 BERT。
8. 第 4 天以后再尝试 BERT。如果模型太慢，就只做特征提取，不做完整微调。
9. 把结果表交给 C，把数据来源和清洗规则交给 D，把 2-3 条样例交给 E。

## 0.3 现成模型和工具怎么选

| 场景 | 推荐选择 | 原因 |
| --- | --- | --- |
| 只想稳稳跑通 | TF-IDF + 逻辑回归 | 快、容易解释、电脑要求低。 |
| 中文文本较多 | `bert-base-chinese` | 常见中文预训练模型，资料多。 |
| 想写得更像深度学习 | 中文 RoBERTa | 可以作为 BERT 的同类改进模型介绍。 |
| 没有 GPU | 冻结预训练模型，只提取文本向量 | 不训练大模型主体，速度更可控。 |
| 效果不好 | 保留结果，重点写错误案例和局限性 | 探索性项目允许效果一般，但不能说不清楚。 |

## 0.4 失败时怎么办

| 问题 | 保底处理 |
| --- | --- |
| 找不到中文数据 | 用英文公开数据先跑流程，报告中说明后续可替换中文数据。 |
| 数据量太少 | 做演示级实验，重点展示流程、字段、模型和评估。 |
| BERT 跑不动 | 退回 TF-IDF，把 BERT 写成可选扩展。 |
| 结果很差 | 检查标签是否混乱，再用错误案例解释难点。 |
| 不懂指标 | 先记住 F1 值越高越好，精确率看误报，召回率看漏报。 |

## 0.5 每天做到什么程度算合格

| 时间 | 合格线 |
| --- | --- |
| 第 1 天 | 字段表和标签定义写清楚。 |
| 第 2 天 | 至少有一份可训练 CSV 或 JSONL 样例。 |
| 第 3 天 | TF-IDF 基线能跑出指标。 |
| 第 4-5 天 | BERT 能跑通，或明确退回 TF-IDF 保底。 |
| 第 6 天 | 有错误案例和行为特征说明。 |
| 第 7 天 | C、D、E 都能拿到你的材料。 |

## 0.6 可以直接写进报告/PPT 的说法

本项目首先采用 TF-IDF + 逻辑回归作为文本基线模型，用于验证文本内容本身是否包含风险识别信号。在此基础上，进一步尝试使用中文预训练语言模型提取文本语义特征，以增强模型对谐音、变体表达和上下文含义的理解。由于项目周期有限，若深度模型训练效果不稳定，则以基线模型结果和错误案例分析作为主要交付。

## 1. 你最终要交付什么

| 交付物 | 最晚时间 | 验收标准 |
| --- | --- | --- |
| 数据字段表 | 第 1 天 | 说明每一列的含义、类型、是否可为空、是否需要脱敏。 |
| 数据来源清单 | 第 2 天 | 至少列出 2 个公开数据来源，说明是否采用、原因和风险。 |
| 清洗后样例数据 | 第 2 天 | 至少能给 C 一份可读取的 CSV 或 JSONL 样例。 |
| 文本基线模型结果 | 第 3 天 | 能输出准确率、精确率、召回率、F1 值。 |
| BERT/中文 RoBERTa 文本分支结果 | 第 4-5 天 | 能独立输入文本并输出预测概率或文本向量。 |
| 用户行为特征表 | 第 5 天 | 至少包含发帖频率、链接数、重复比例、互动比等可解释字段。 |

## 2. 第 1 天：确定数据字段和标签

### 操作步骤

1. 和全组确认本项目识别对象：垃圾广告、诈骗内容、异常账号或有害内容。
2. 确认标签数量。短周期项目优先做二分类：`normal` 和 `risk`。
3. 建立最小字段表：

```text
sample_id
text
image_path
behavior_features
label
source
split
```

4. 明确哪些字段可以为空。例如没有图片时 `image_path` 为空，但 `text` 和 `label` 尽量不为空。
5. 写一份“标签定义说明”，说明什么算正常、什么算风险。

### 当天检查清单

- 已确定二分类或多分类。
- 已明确每个字段的含义。
- 已和 B 确认 `image_path` 的格式。
- 已和 C 确认最终数据文件格式。
- 已和 D 确认数据来源说明可以写进报告。

## 3. 第 2 天：找公开数据集并清洗脱敏

### 推荐找数路线

1. 先找公开数据集平台：Hugging Face Datasets、Kaggle、GitHub 数据仓库、论文附录。
2. 搜索关键词使用：

```text
social media spam dataset
fraud text dataset
harmful content detection dataset
Chinese spam text dataset
multimodal spam dataset
```

3. 如果找不到完全匹配的数据，就先用“文本风险识别数据 + 图片样例数据 + 行为特征模拟表”的组合方式跑通流程。
4. 爬虫只作为补充，只采集公开页面，不登录、不保存 cookie、不采集私信和个人敏感信息。

### 清洗规则

| 问题 | 处理方式 |
| --- | --- |
| 文本为空 | 删除或标记为缺失，不参与文本模型训练。 |
| 文本过长 | 截断到模型最大长度，常用 128 或 256。 |
| 重复样本 | 按文本哈希或 `sample_id` 去重。 |
| 个人信息 | 删除手机号、邮箱、主页链接、账号名、二维码信息。 |
| 标签混乱 | 先合并成 `normal` 和 `risk`，保证项目能跑通。 |

### 当天检查清单

- 至少准备一份可训练样例数据。
- 每条数据有 `sample_id` 和 `label`。
- 原始数据不提交到 GitHub。
- 脱敏规则已写成文字，交给 D。

## 4. 第 3 天：跑通文本基线模型

### 目标

先用简单方法证明文本内容有识别信号，不要一开始就卡在大模型上。

### 推荐流程

```text
读取清洗数据
-> 分出训练集、验证集、测试集
-> TF-IDF 提取文本特征
-> 逻辑回归分类
-> 输出指标和错误样例
```

### 建议记录的结果

| 项目 | 说明 |
| --- | --- |
| 文本总样本数 | 写清训练、验证、测试数量。 |
| 类别比例 | 写清正常样本和风险样本比例。 |
| 主要指标 | 准确率、精确率、召回率、F1 值。 |
| 错误案例 | 找 3 条误判样本，说明可能原因。 |

### 当天检查清单

- 文本基线模型能独立跑完。
- 指标表已交给 C。
- 错误样例已交给 D。
- 发现类别不均衡时已记录比例。

## 5. 第 4-5 天：跑通 BERT 或中文 RoBERTa

### 选择规则

| 条件 | 推荐方案 |
| --- | --- |
| 数据很少、电脑较慢 | 冻结 BERT 主体，只提取文本向量。 |
| 有 GPU 或样本较多 | 微调 BERT/中文 RoBERTa 分类头。 |
| 时间不够 | 保留 TF-IDF 基线，把 BERT 作为扩展结果。 |

### 推荐流程

```text
文本
-> 分词器
-> BERT/中文 RoBERTa
-> [CLS] 向量
-> 分类层或导出文本向量
-> 交给 C 做评估或融合
```

### 你要交给 B/C 的内容

| 文件或材料 | 用途 |
| --- | --- |
| 文本向量维度说明 | B 做多模态拼接时需要知道维度。 |
| 文本预测概率 | C 做单模态评估和后期融合对比。 |
| 文本模型参数 | D 写报告方法章节时需要引用。 |
| 文本错误案例 | E 准备答辩问题时需要解释。 |

## 6. 第 5 天：补充用户行为特征

用户行为特征不要做复杂，先做可解释统计：

| 特征 | 计算思路 |
| --- | --- |
| 发帖频率 | 单位时间内发帖数量。 |
| 链接数量 | 文本中 URL 数量。 |
| 重复内容比例 | 同一用户重复文本或相似文本比例。 |
| 粉丝关注比 | 粉丝数除以关注数，缺失时填 0 或中位数。 |
| 互动比例 | 点赞、评论、转发除以粉丝数。 |
| 敏感词数量 | 命中广告、诈骗、导流词的数量。 |

交给 C 时要说明每个字段是否已经标准化。如果没有标准化，提醒 C 在融合前统一处理。

## 7. 第 6 天：做错误分析和消融准备

你需要准备：

1. 文本模型最容易误判的样本。
2. 文本模型比图像模型更有效的样本。
3. 文本模型失效但图片或行为可能有效的样本。
4. 类别不均衡、文本太短、黑话谐音、截图文字等问题说明。

这些内容会直接进入报告的“错误分析”和“局限性”部分。

## 8. 第 7 天：交接包

### 交给 C

- 清洗后数据路径说明。
- 字段说明。
- 文本模型指标表。
- 文本向量或预测概率说明。
- 行为特征字段说明。

### 交给 D

- 数据来源和脱敏说明。
- 文本模型方法说明。
- 训练参数说明。
- 3-5 个错误案例。
- 300-500 字技术解释。

### 交给 E

- 2-3 张适合放 PPT 的数据样例图。
- 一句话说明文本模型为什么必要。
- 老师可能问到的数据合规问题和回答要点。

## 9. 常见问题和补救方案

| 问题 | 补救方案 |
| --- | --- |
| 找不到完全匹配的多模态数据 | 先用公开文本数据跑主线，图片和行为特征用小样例补齐流程。 |
| 类别严重不均衡 | 使用 F1 值作为主要指标，记录类别比例，训练时使用类别权重。 |
| BERT 跑得太慢 | 冻结模型，只导出文本向量；必要时退回 TF-IDF 基线。 |
| 数据来源不方便公开 | 只写来源类别和采集规则，不提交原始数据和敏感字段。 |

## 10. 每天结束前自查

- 今天的输出是否能被 C 直接读取。
- 今天的结论是否能被 D 写进报告。
- 今天的图表是否能被 E 放进 PPT。
- 是否有任何隐私、账号、密钥或未脱敏数据进入仓库。
- 是否把无法完成的风险及时告诉 C、D、E。
