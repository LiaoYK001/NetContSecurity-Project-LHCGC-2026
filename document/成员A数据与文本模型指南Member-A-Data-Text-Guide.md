# 成员 A 数据与文本模型指南

适用角色：数据负责人兼文本模型负责人  
目标周期：第 1 天到第 7 天  
核心目标：把“可训练、可复现、可解释”的文本与行为数据准备好，并跑通文本基线模型和 BERT/中文 RoBERTa 文本分支。

## 1. 你最终要交付什么

| 交付物 | 最晚时间 | 验收标准 |
| --- | --- | --- |
| 数据字段表 | 第 1 天 | 说明每一列的含义、类型、是否可为空、是否需要脱敏。 |
| 数据来源清单 | 第 2 天 | 至少列出 2 个公开数据来源，说明是否采用、原因和风险。 |
| 清洗后样例数据 | 第 2 天 | 至少能给 C 一份可读取的 CSV 或 JSONL 样例。 |
| 文本基线模型结果 | 第 3 天 | 能输出准确率、精确率、召回率、F1 值。 |
| BERT/中文 RoBERTa 文本分支结果 | 第 4-5 天 | 能独立输入文本并输出预测概率或文本向量。 |
| 用户行为特征表 | 第 5 天 | 至少包含发帖频率、链接数、重复比例、互动比等可解释字段。 |
| 第 7 天交接材料 | 第 7 天 | 数据说明、文本模型说明、关键指标、错误案例、300-500 字技术解释。 |

## 2. 第 1 天：确定数据字段和标签

### 操作步骤

1. 和全组确认本项目识别对象：垃圾广告、诈骗内容、异常账号或有害内容。
2. 确认标签数量。短周期项目优先做二分类：`normal` 和 `risk`。
3. 建立最小字段表：

```text
sample_id
text
image_path
behavior_features
label
source
split
```

4. 明确哪些字段可以为空。例如没有图片时 `image_path` 为空，但 `text` 和 `label` 尽量不为空。
5. 写一份“标签定义说明”，说明什么算正常、什么算风险。

### 当天检查清单

- 已确定二分类或多分类。
- 已明确每个字段的含义。
- 已和 B 确认 `image_path` 的格式。
- 已和 C 确认最终数据文件格式。
- 已和 D 确认数据来源说明可以写进报告。

## 3. 第 2 天：找公开数据集并清洗脱敏

### 推荐找数路线

1. 先找公开数据集平台：Hugging Face Datasets、Kaggle、GitHub 数据仓库、论文附录。
2. 搜索关键词使用：

```text
social media spam dataset
fraud text dataset
harmful content detection dataset
Chinese spam text dataset
multimodal spam dataset
```

3. 如果找不到完全匹配的数据，就先用“文本风险识别数据 + 图片样例数据 + 行为特征模拟表”的组合方式跑通流程。
4. 爬虫只作为补充，只采集公开页面，不登录、不保存 cookie、不采集私信和个人敏感信息。

### 清洗规则

| 问题 | 处理方式 |
| --- | --- |
| 文本为空 | 删除或标记为缺失，不参与文本模型训练。 |
| 文本过长 | 截断到模型最大长度，常用 128 或 256。 |
| 重复样本 | 按文本哈希或 `sample_id` 去重。 |
| 个人信息 | 删除手机号、邮箱、主页链接、账号名、二维码信息。 |
| 标签混乱 | 先合并成 `normal` 和 `risk`，保证项目能跑通。 |

### 当天检查清单

- 至少准备一份可训练样例数据。
- 每条数据有 `sample_id` 和 `label`。
- 原始数据不提交到 GitHub。
- 脱敏规则已写成文字，交给 D。

## 4. 第 3 天：跑通文本基线模型

### 目标

先用简单方法证明文本内容有识别信号，不要一开始就卡在大模型上。

### 推荐流程

```text
读取清洗数据
-> 分出训练集、验证集、测试集
-> TF-IDF 提取文本特征
-> 逻辑回归分类
-> 输出指标和错误样例
```

### 建议记录的结果

| 项目 | 说明 |
| --- | --- |
| 文本总样本数 | 写清训练、验证、测试数量。 |
| 类别比例 | 写清正常样本和风险样本比例。 |
| 主要指标 | 准确率、精确率、召回率、F1 值。 |
| 错误案例 | 找 3 条误判样本，说明可能原因。 |

### 当天检查清单

- 文本基线模型能独立跑完。
- 指标表已交给 C。
- 错误样例已交给 D。
- 发现类别不均衡时已记录比例。

## 5. 第 4-5 天：跑通 BERT 或中文 RoBERTa

### 选择规则

| 条件 | 推荐方案 |
| --- | --- |
| 数据很少、电脑较慢 | 冻结 BERT 主体，只提取文本向量。 |
| 有 GPU 或样本较多 | 微调 BERT/中文 RoBERTa 分类头。 |
| 时间不够 | 保留 TF-IDF 基线，把 BERT 作为扩展结果。 |

### 推荐流程

```text
文本
-> 分词器
-> BERT/中文 RoBERTa
-> [CLS] 向量
-> 分类层或导出文本向量
-> 交给 C 做评估或融合
```

### 你要交给 B/C 的内容

| 文件或材料 | 用途 |
| --- | --- |
| 文本向量维度说明 | B 做多模态拼接时需要知道维度。 |
| 文本预测概率 | C 做单模态评估和后期融合对比。 |
| 文本模型参数 | D 写报告方法章节时需要引用。 |
| 文本错误案例 | E 准备答辩问题时需要解释。 |

## 6. 第 5 天：补充用户行为特征

用户行为特征不要做复杂，先做可解释统计：

| 特征 | 计算思路 |
| --- | --- |
| 发帖频率 | 单位时间内发帖数量。 |
| 链接数量 | 文本中 URL 数量。 |
| 重复内容比例 | 同一用户重复文本或相似文本比例。 |
| 粉丝关注比 | 粉丝数除以关注数，缺失时填 0 或中位数。 |
| 互动比例 | 点赞、评论、转发除以粉丝数。 |
| 敏感词数量 | 命中广告、诈骗、导流词的数量。 |

交给 C 时要说明每个字段是否已经标准化。如果没有标准化，提醒 C 在融合前统一处理。

## 7. 第 6 天：做错误分析和消融准备

你需要准备：

1. 文本模型最容易误判的样本。
2. 文本模型比图像模型更有效的样本。
3. 文本模型失效但图片或行为可能有效的样本。
4. 类别不均衡、文本太短、黑话谐音、截图文字等问题说明。

这些内容会直接进入报告的“错误分析”和“局限性”部分。

## 8. 第 7 天：交接包

### 交给 C

- 清洗后数据路径说明。
- 字段说明。
- 文本模型指标表。
- 文本向量或预测概率说明。
- 行为特征字段说明。

### 交给 D

- 数据来源和脱敏说明。
- 文本模型方法说明。
- 训练参数说明。
- 3-5 个错误案例。
- 300-500 字技术解释。

### 交给 E

- 2-3 张适合放 PPT 的数据样例图。
- 一句话说明文本模型为什么必要。
- 老师可能问到的数据合规问题和回答要点。

## 9. 常见问题和补救方案

| 问题 | 补救方案 |
| --- | --- |
| 找不到完全匹配的多模态数据 | 先用公开文本数据跑主线，图片和行为特征用小样例补齐流程。 |
| 类别严重不均衡 | 使用 F1 值作为主要指标，记录类别比例，训练时使用类别权重。 |
| BERT 跑得太慢 | 冻结模型，只导出文本向量；必要时退回 TF-IDF 基线。 |
| 数据来源不方便公开 | 只写来源类别和采集规则，不提交原始数据和敏感字段。 |

## 10. 每天结束前自查

- 今天的输出是否能被 C 直接读取。
- 今天的结论是否能被 D 写进报告。
- 今天的图表是否能被 E 放进 PPT。
- 是否有任何隐私、账号、密钥或未脱敏数据进入仓库。
- 是否把无法完成的风险及时告诉 C、D、E。
