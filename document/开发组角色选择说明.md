# 开发组角色选择说明：A / B / C 怎么选？

适用场景：队员还没决定选哪个开发角色，想搞清楚**每个角色到底做什么、难不难、适不适合自己**。

读完本文后，你应该能回答三个问题：

1. 成员 A/B/C 每天具体在干什么？（用大白话说）
2. 我做这个角色需要会什么？不会怎么办？
3. 我能做吗？

> 本文是**选角色用的**。选完之后，再读各自的个人指南（`成员X...指南.md`）和 [开发组协作指南](开发组协作指南.md)。

---

## 0. 一句话区分三个角色

| 角色       | 一句话                | 类比                                                                |
| ---------- | --------------------- | ------------------------------------------------------------------- |
| **成员 A** | 管数据 + 跑文本模型   | 厨房里**备菜的人**：菜不洗好，别人没法下锅                          |
| **成员 B** | 管图片 + 拼多模态融合 | 厨房里**炒菜的人**：把 A 备好的文本菜、自己的图片菜、行为调料一起炒 |
| **成员 C** | 管评估 + 画图 + 打包  | 厨房里**品控和摆盘的人**：尝味道、打分、拍照、端出去                |

## 1. 成员 A：数据 + 文本模型 + 行为特征

### 1.1 我每天在做什么？（白话版）

```
第1天：和全组定好标签（正常/风险），定好数据表有哪几列
第2天：去 HuggingFace/Kaggle 搜公开数据集，下载 CSV，清洗脏数据（去重、删隐私、统一格式）
第3天：用最简单的 TF-IDF + 逻辑回归 跑一个文本分类，拿到准确率/F1
第4天：如果电脑跑得动，用 BERT/中文 RoBERTa 提取文本特征向量
第5天：算几个行为特征（比如发帖频率、链接数量、粉丝比）
第6天：配合 B/C 做消融实验（去掉某个部分看效果变化），找几个模型判错的样本
第7天：整理数据说明、模型说明、写 300-500 字技术解释，全部交给 C
```

### 1.2 具体要写的代码/做的事

| 任务                             | 用什么工具                                                                                    | 难度        |
| -------------------------------- | --------------------------------------------------------------------------------------------- | ----------- |
| 找公开数据集                     | HuggingFace Datasets / Kaggle 搜索下载                                                        | ⭐ 搜就完了 |
| 清洗数据（去重、脱敏、统一格式） | `pandas` 读 CSV、`drop_duplicates()`、正则删手机号                                            | ⭐⭐        |
| TF-IDF + 逻辑回归文本分类        | `sklearn.feature_extraction.text.TfidfVectorizer` + `sklearn.linear_model.LogisticRegression` | ⭐⭐        |
| BERT/中文 RoBERTa 提取文本向量   | `transformers.AutoModel` + `AutoTokenizer`（网上复制现成代码改）                              | ⭐⭐⭐      |
| 计算行为特征                     | `pandas` 统计：count、mean、groupby                                                           | ⭐⭐        |
| 写技术解释                       | Markdown / Word                                                                               | ⭐          |

### 1.3 什么时候最忙？

```
第1天 ████████░░ 中等（定规范、搜数据）
第2天 ██████████ 最忙（下载数据 + 清洗，容易出幺蛾子）
第3天 ████████░░ 中等（跑 TF-IDF 基线，代码不长）
第4天 █████████░ 偏忙（如果 BERT 跑不动要降级）
第5天 ██████░░░░ 较轻松（算行为特征，数据已有）
第6天 ████████░░ 中等（配合消融实验）
第7天 ██████░░░░ 较轻松（写技术解释，整理材料）
```

### 1.4 适合什么样的人？

| 适合 ✅                      | 不太适合 ❌      |
| ---------------------------- | ---------------- |
| 会用 pandas 处理 CSV         | 从没用过 Python  |
| 愿意花时间搜公开数据         | 不想碰数据清洗   |
| 不怕写"数据处理脚本"这类代码 | 只想做纯模型训练 |
| 细心（数据错了全组翻车）     | 粗心、不喜欢检查 |

### 1.5 最大的风险是什么？

**找不到合适的公开数据。** 如果第 2 天结束还没拿到数据集，B 和 C 都等着。→ 保底方案：用英文公开 spam 数据集先跑通流程，报告中说明后续可换中文数据。

## 2. 成员 B：图像模型 + 多模态融合

### 2.1 我每天在做什么？（白话版）

```
第1天：和 A 定好图片路径怎么写（比如 "data/raw/img/001.jpg"），和 C 定好图像向量存哪里
第2天：写好图片读取、缩放、归一化的代码框架，先用几张样例跑通
第3天：用预训练 ResNet18/50 提取所有图片的特征向量（不用自己训练！）
第4天：收尾图像单模态结果，把图片预测交给 C
第5天：最关键的一天！拿 A 的文本向量 + 自己的图像向量 + A 的行为特征，拼一起接 MLP 分类器
第6天：配合做消融实验（仅文本 / 仅图像 / 文本+图像 / 全部），分析图像贡献多少
第7天：画模型结构图，写 300-500 字技术解释，交给 C
```

### 2.2 具体要写的代码/做的事

| 任务                                     | 用什么工具                                     | 难度   |
| ---------------------------------------- | ---------------------------------------------- | ------ |
| 图片读取、resize、归一化                 | `PIL.Image` / `torchvision.transforms`         | ⭐⭐   |
| ResNet18/50 提取图像特征（冻结，不训练） | `torchvision.models.resnet18(pretrained=True)` | ⭐⭐   |
| 把文本向量 + 图像向量 + 行为特征拼一起   | `torch.cat()` 或 `np.concatenate()`            | ⭐⭐   |
| MLP 分类器                               | `torch.nn.Linear` 几层全连接                   | ⭐⭐   |
| 训练融合模型                             | 标准训练循环（optimizer、loss、epoch）         | ⭐⭐⭐ |
| 画模型结构图                             | Mermaid / draw.io / PPT 手画                   | ⭐     |

### 2.3 什么时候最忙？

```
第1天 ████░░░░░░ 轻松（只定规范，不写代码）
第2天 ██████░░░░ 较轻松（写预处理框架）
第3天 ██████████ 繁忙（跑 ResNet 提取所有图片）
第4天 ████████░░ 中等（收尾图像）
第5天 ██████████ 最忙！！（融合是最核心的一天，依赖 A 交付）
第6天 █████████░ 偏忙（消融实验可能反复调）
第7天 ██████░░░░ 较轻松（画图 + 写解释）
```

### 2.4 适合什么样的人？

| 适合 ✅                     | 不太适合 ❌              |
| --------------------------- | ------------------------ |
| 用过 PyTorch 或至少看过教程 | 完全不知道什么是神经网络 |
| 对"把不同东西拼一起"有耐心  | 害怕调试维度不匹配的报错 |
| 能接受"等别人交付才能继续"  | 急性子，不想等 A 的产出  |
| 愿意写模型结构说明          | 只写代码不写文档         |

### 2.5 最大的风险是什么？

**等 A 的文本向量，但 A 可能卡住。** B 的第 5 天能否做融合，完全取决于 A 在第 4-5 天有没有交出文本向量和行为特征。

→ 保底方案：如果 A 卡住了，B 先用"分数加权融合"（文本概率 × 0.5 + 图像概率 × 0.3 + 行为概率 × 0.2），不需要拼接向量，也不等 MLP 训练。

## 3. 成员 C：系统集成 + 评测 + 交接

### 3.1 我每天在做什么？（白话版）

```
第1天：定好全组的目录结构、文件命名、命令格式（比如 predict CSV 必须5列）
第2天：建实验记录表（Markdown 或 CSV），准备好评估函数
第3天：拿 A 的文本基线结果，用 scikit-learn 算准确率/F1/混淆矩阵
第4天：汇总 A 的文本 + B 的图像的单模态结果，确认两边的测试集一样
第5天：评估 B 的融合模型，把文本/图像/融合放在同一张表里对比
第6天：最忙的一天！画混淆矩阵、ROC 曲线、消融柱状图，整理错误案例
第7天：把全组的材料打包成交接包，让报告组 D/E 能直接看懂
```

### 3.2 具体要写的代码/做的事

| 任务                         | 用什么工具                                             | 难度 |
| ---------------------------- | ------------------------------------------------------ | ---- |
| 算准确率、精确率、召回率、F1 | `sklearn.metrics.classification_report`                | ⭐⭐ |
| 画混淆矩阵                   | `sklearn.metrics.confusion_matrix` + `seaborn.heatmap` | ⭐⭐ |
| 画 ROC/PR 曲线               | `sklearn.metrics.roc_curve` + `matplotlib`             | ⭐⭐ |
| 画消融实验柱状图             | `matplotlib.pyplot.bar`                                | ⭐⭐ |
| 汇总 CSV 指标表              | `pandas` 读多份预测，合并对比                          | ⭐⭐ |
| 写复现命令                   | Markdown / 注释                                        | ⭐   |
| 打包交接包                   | 整理文件 + 写说明                                      | ⭐   |

### 3.3 什么时候最忙？

```
第1天 ████░░░░░░ 轻松（定规范，不需要写太多代码）
第2天 ████░░░░░░ 轻松（建好记录表模板即可）
第3天 ██████░░░░ 较轻松（拿 A 的结果练手算指标）
第4天 ████████░░ 中等（开始同时跟进 A 和 B 的进度）
第5天 █████████░ 偏忙（等 B 的融合结果，马上评估对比）
第6天 ██████████ 最忙！！（画所有图表、做消融实验、整理错误案例）
第7天 ██████████ 最忙！！（打包交接，D/E 能不能开工全看你的包）
```

### 3.4 适合什么样的人？

| 适合 ✅                           | 不太适合 ❌            |
| --------------------------------- | ---------------------- |
| 擅长用 scikit-learn 和 matplotlib | 不想画图、不想做表     |
| 有条理，能同时跟进 2-3 个人的进度 | 只关心自己的一亩三分地 |
| 能把技术结果翻译成"人话"          | 觉得只要代码跑通就行   |
| 愿意做项目总管                    | 不想催别人交材料       |

### 3.5 最大的风险是什么？

**A 或 B 交付的格式不一致，无法统一评估。** 比如 A 的 CSV 列名叫 `label`，B 的叫 `true_label`，C 就要手动对齐。

→ C 必须在第 1 天就**强制公布统一格式**（5 列：`sample_id`, `true_label`, `pred_label`, `risk_prob`, `model_name`），谁不按格式来就喊停。

| -------------------------------------- | -------- |

| 我知道精确率、召回率、F1 值是什么      | /3       |

| **C 总分**                             | **/12**  |

### 怎么看结果？

- 哪个角色的分最高，就优先选哪个。
- 如果两个角色得分接近，优先选**你更感兴趣的那个**。
- 如果三个都低（都 < 6 分），不用慌——这是选修课项目，三份个人指南里都有"白话版"和"手把手步骤"，跟着做就行。建议优先选 **C**（代码量最小、最适合边学边做）。

## 6. 常见担忧（Q&A）

### Q1: 我不会 PyTorch，能做 B 吗？

能做。保底方案是"冻结 ResNet + 分数加权融合"，差不多 30 行代码。标准方案才需要写训练循环。个人指南里给了完整的代码框架。

### Q2: 我不知道 BERT 是什么，能做 A 吗？

能做。保底方案是 TF-IDF + 逻辑回归，这是 sklearn 两行代码的事。BERT 是标准方案的可选项，跑不动就退回到保底。

### Q3: C 是不是最轻松？就画几张图？

不是。C 最忙的时候是 Day 6-7——要画所有图表、做消融实验、整理错误案例、打交接包。而且 C 承担"总管"角色，要催 A 和 B 按时交材料。但 C 确实不需要写模型训练代码。

### Q4: 我选 A，但我不会找数据怎么办？

个人指南里已经给了搜索关键词（`Chinese spam text dataset` 等），也给了常用平台（HuggingFace、Kaggle）。如果实在找不到匹配的中文数据，用英文公开数据先跑通流程，报告里写明原因即可。

### Q5: 三个人可以互换任务吗？

可以！比如 A 的 TF-IDF 跑完后去帮 B 写 MLP 分类器，或者 B 帮 C 画混淆矩阵。角色分工是**责任边界**，但代码可以互相帮忙写。只要各自的个人指南里的"交付物"按时交给 C 就行。

### Q6: 我选错了怎么办？

第 1-2 天内可以换。第 3 天以后不建议大换，但可以互帮互助。重点是第 7 天全组交出完整交付物，谁写的代码不是最重要的。
