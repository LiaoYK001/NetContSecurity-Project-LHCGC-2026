# 成员 E PPT 与答辩指南

适用角色：PPT 制作、讲稿组织和答辩主讲  
目标周期：第 1 天到第 10 天，必要时延长到第 12 天  
核心目标：把报告中的研究问题、方法、实验和结论压缩成一套能讲清楚、能答辩、能展示小组成果的 PPT。

## 1. 你最终要交付什么

| 交付物 | 最晚时间 | 验收标准 |
| --- | --- | --- |
| PPT 目录 | 第 1 天 | 能覆盖问题、方法、数据、实验、结论和分工。 |
| PPT 初版 | 第 8 天 | 至少完成 60%，等待最终实验图表替换。 |
| PPT 完整版 | 第 9 天 | 页面完整，图表和报告一致。 |
| 讲稿 | 第 9-10 天 | 能在规定时间内讲完。 |
| 问答表 | 第 10 天 | 至少准备 10 个老师可能追问的问题。 |
| 模拟答辩记录 | 第 10-12 天 | 记录超时点、卡顿点和被问住的问题。 |

## 2. PPT 推荐结构

```text
1 研究背景与问题
2 任务定义
3 数据来源与预处理
4 系统总体流程
5 文本、图像、行为特征方法
6 多模态融合方法
7 实验结果与消融分析
8 错误案例与局限性
9 总结与展望
10 成员分工
```

如果答辩时间很短，压缩成 8 页左右；如果时间较长，可以扩展到 12-15 页。

## 3. 第 1 天：建立答辩主线

### 你要讲清楚的一句话

本项目针对社交平台垃圾广告、诈骗和有害内容难以仅靠文本识别的问题，尝试结合文本、图像和用户行为特征，构建一个可复现的多模态风险识别流程。

### 当天操作

1. 确认答辩时长。
2. 建立 PPT 目录。
3. 和 D 对齐报告目录。
4. 先做空白页框架，不等实验结果。
5. 列出需要开发组提供的图表。

### 当天检查清单

- PPT 有清晰开头和结尾。
- 每一页知道要讲什么。
- 已列出要向 A/B/C 要的素材。
- 已确认老师是否要求现场演示。

## 4. 第 2-4 天：先做背景和方法页

### 背景页

只讲三个点：

1. 有害内容会影响平台安全和用户体验。
2. 单纯文本检测容易被谐音、截图、变体图片绕过。
3. 多模态方法能利用更多信号。

### 方法页

建议画一张总流程图：

```text
公开数据
-> 文本清洗和文本模型
-> 图片预处理和图像模型
-> 用户行为特征
-> 多模态融合
-> 风险识别结果
```

不要在 PPT 堆太多公式。公式和细节放报告，PPT 重点讲逻辑。

## 5. 第 5-7 天：等结果时先准备页面占位

你可以先做这些页面：

| 页面 | 先放什么 |
| --- | --- |
| 数据样例 | 放脱敏样例，不放真实账号。 |
| 模型结构 | 先放流程图，等 B 给最终结构图。 |
| 指标页 | 先放空表格，等 C 给最终指标。 |
| 消融实验页 | 先放表格模板，等 C 给结果。 |
| 错误案例页 | 先放版式，等 A/B/C 给样例。 |

第 7 天开发组交接后，你要快速替换占位内容。

## 6. 第 8-9 天：完成 PPT 和讲稿

### 页面设计原则

| 原则 | 说明 |
| --- | --- |
| 一页一个结论 | 每页只表达一个核心观点。 |
| 图表少而清楚 | PPT 不放过密表格，复杂表格放报告。 |
| 标题说结论 | 不写“实验结果”，改写成“多模态融合提升了 F1 值”。 |
| 颜色克制 | 不要用过多颜色，重点突出最终方案。 |
| 不泄露隐私 | 图片样例必须脱敏，不能出现真实账号和链接。 |

### 讲稿写法

每页讲稿建议 3 句话：

1. 这一页讲什么。
2. 我们怎么做。
3. 这说明什么。

不要逐字背诵大段文字，重点记住逻辑顺序和关键数字。

## 7. 第 10 天：准备问答

至少准备以下问题：

| 问题 | 回答方向 |
| --- | --- |
| 为什么不用单纯文本模型？ | 文本容易被谐音、图片和变体绕过，多模态能补充信号。 |
| 数据是否合规？ | 使用公开数据集优先，爬虫只作为补充，且不采集隐私和登录后内容。 |
| 为什么选 BERT 和 ResNet？ | 成熟、稳定、可复现，适合短周期课程项目。 |
| 融合方式为什么简单拼接？ | 项目周期短，拼接 + MLP 稳定可解释，便于复现。 |
| 指标为什么看 F1 值？ | 类别可能不均衡，F1 值能综合精确率和召回率。 |
| 模型误判怎么办？ | 用错误案例分析局限性，并提出 OCR、更多数据、注意力融合等改进。 |
| public 仓库如何保护隐私？ | 不提交原始数据、密钥、账号、模型权重，只提交说明和脱敏样例。 |
| 如果没有 GPU 怎么办？ | 使用 TF-IDF、冻结预训练模型和小批量实验，保证流程跑通。 |
| 图像分支效果差是否失败？ | 不失败，消融实验能说明图像信号在当前数据中的贡献和限制。 |
| 后续如何改进？ | 扩大数据、加入 OCR、优化融合方式、接入合规真实平台样本。 |

## 8. 模拟答辩流程

1. 第一次试讲只看是否能讲完。
2. 第二次试讲记录每页用时。
3. 第三次试讲让 A/B/C/D 各问 2 个问题。
4. 把答不上来的问题写进问答表。
5. 最后压缩超时页面，保留关键结论。

## 9. 常见问题和补救方案

| 问题 | 补救方案 |
| --- | --- |
| 实验结果来得太晚 | 先完成所有页面结构，最后只替换数字和图表。 |
| PPT 内容太多 | 每页保留一个结论，细节放报告。 |
| 讲稿超时 | 删除背景冗余内容，重点讲方法和结果。 |
| 被问到技术细节 | 先回答核心逻辑，再请对应开发成员补充。 |
| 老师质疑数据合规 | 强调公开数据集优先、脱敏、public 仓库不提交原始数据。 |

## 10. 每天结束前自查

- PPT 是否和报告结论一致。
- 每张图是否有来源。
- 是否有页面文字过多。
- 是否能在规定时间内讲完。
- 是否准备了老师可能追问的问题。
- 是否没有出现真实账号、链接、二维码、手机号等隐私信息。

## 11. 最终提交清单

- PPT 终稿。
- 讲稿。
- 问答表。
- 演示备用文件。
- 成员分工页。
- 模拟答辩记录。
