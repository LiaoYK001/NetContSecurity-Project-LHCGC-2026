# 06 答辩问答表

## 1. 项目目标类

| 问题 | 建议回答 |
| --- | --- |
| 你们项目到底做什么？ | 我们做多模态社交网络内容风控，用文本、图片和用户行为共同判断内容是否存在风险。 |
| 为什么叫多模态？ | 因为模型不只看文本，还看图片视觉特征和行为结构化特征。 |
| 为什么不只做文本分类？ | 风险信息可能藏在图片中，或者通过异常行为体现，只看文本不够完整。 |

## 2. 数据类

| 问题 | 建议回答 |
| --- | --- |
| 数据从哪里来？ | 默认使用公开 EANN-KDD18 Weibo 数据集，并在本地转换成项目 CSV。 |
| 标签是什么？ | 二分类，`normal` 表示正常内容，`risk` 表示风险内容。 |
| public 仓库为什么没有完整数据？ | 为了保护隐私和避免提交大体积数据，仓库只提交代码、说明、脱敏样例和 public-safe demo 结果。 |
| 怎么保证文本、图片、行为对应同一条样本？ | 用 `sample_id` 作为对齐钥匙，所有特征表都按它合并。 |

## 3. 模型类

| 问题 | 建议回答 |
| --- | --- |
| 为什么用 TF-IDF？ | 它是快速稳定的文本保底方案，便于确认流程和提供可解释基线。 |
| BERT 在哪里用？ | BERT 用在文本分支，把每条文本编码成 768 维语义向量，再参与融合。 |
| 为什么不微调 BERT？ | 项目周期短，冻结 BERT 更稳定、算力要求更低，也更容易复现。 |
| 为什么用 ResNet18？ | ResNet18 是成熟的预训练图像模型，适合快速提取 512 维视觉特征。 |
| 行为特征是什么？ | 包括认证状态、转发、评论、点赞、粉丝、关注、发帖数、文本长度、链接数等结构化字段。 |
| 融合怎么做？ | 把 BERT 文本向量、ResNet 图像向量和行为向量按 `sample_id` 拼接，再训练分类器。 |

## 4. 实验结果类

| 问题 | 建议回答 |
| --- | --- |
| 最终方案效果如何？ | 完整三模态方案 F1 为 0.9640，ROC AUC 为 0.9923。 |
| 为什么 behavior_only 最高？ | 当前数据集上行为特征区分度很强，但可能存在数据分布偏差，需要谨慎解释。 |
| 既然行为特征最高，为什么还要多模态？ | 真实场景不能只依赖单一行为字段。三模态方案更符合项目目标，也更稳健可解释。 |
| 图像分支效果弱是不是失败？ | 不是。消融实验正是为了发现不同模态贡献。图像在当前数据中单独弱，但可作为辅助信号。 |
| 为什么看 F1？ | F1 同时考虑精确率和召回率，适合风险识别这类既怕误报又怕漏报的任务。 |
| ROC AUC 说明什么？ | 它说明模型整体区分 normal 和 risk 的能力，越接近 1 越好。 |

## 5. 隐私和复现类

| 问题 | 建议回答 |
| --- | --- |
| GitHub 上公开了什么？ | 公开代码、教程、字段样例、汇总指标和图表 demo。 |
| GitHub 上没有公开什么？ | 不公开完整数据、原文、样本级预测、错误案例、模型权重、cookie、token。 |
| 别人能复现吗？ | 可以复现流程。要复现最终指标，需要下载同源公开数据或使用课程私下分发的 processed 数据。 |
| 为什么 outputs/demo 可以公开？ | 它只包含汇总指标和图表，不含原文、真实样本 ID、用户链接或预测明细。 |

## 6. 改进方向类

| 问题 | 建议回答 |
| --- | --- |
| 后续怎么提升？ | 可以扩大数据规模、加入 OCR、尝试注意力融合、增加更严格的跨数据集验证。 |
| 如果换数据集怎么办？ | 只要转换成 `dataset_v1.csv`、`text_samples.csv`、`behavior_features.csv` 三张表，就能复用流程。 |
| 如何降低行为特征偏差？ | 可以做跨时间、跨事件、跨平台测试，并检查行为字段是否携带标签泄漏。 |

## 7. 被追问时的稳妥说法

> 本项目是选修课探索性工程项目，我们重点不是证明某个模型达到论文级最优，而是完整跑通了数据处理、文本/图像/行为特征提取、多模态融合、消融实验、指标图表和隐私合规展示。对于行为特征偏强、图像分支较弱等现象，我们没有回避，而是通过消融实验和局限性分析把它们解释清楚。
