# 01 项目总览与答辩主线

## 1. 5 分钟读懂项目

本项目做的是“多模态社交网络内容风控”。白话说，就是不要只看一条微博写了什么，还要看它有没有配图、图片大概是什么视觉特征、发布者或内容相关行为是否异常。最后把这些信息合在一起，判断内容是 `normal` 还是 `risk`。

项目不是追求论文级最强模型，而是课程探索性工程项目。评分重点应放在：

1. 数据来源清楚。
2. 文本、图像、行为三类信息都被量化。
3. 多模态融合流程能跑通。
4. 指标和图表能解释。
5. public 仓库没有泄露隐私。

## 2. 为什么不只看文本

只看文本有三个问题：

| 问题 | 例子 | 本项目怎么处理 |
| --- | --- | --- |
| 文本会被规避 | 谐音、拆字、变体表达 | 用 BERT 提取上下文语义 |
| 风险内容可能藏在图片里 | 截图、广告图、二维码风格 | 用 ResNet 提取图片视觉特征 |
| 内容看似正常但行为异常 | 频繁发帖、链接多、互动异常 | 加入用户行为特征 |

所以本项目的核心主线是：

```text
文本看语义
图片看视觉
行为看异常
融合后综合判断风险
```

## 3. 一句话答辩主线

> 我们围绕社交平台内容安全问题，构建了一个文本、图像和用户行为联合建模的风险识别流程。项目使用公开 Weibo 数据集，先把原始数据转换成统一 CSV，再分别提取 BERT 文本向量、ResNet 图像向量和行为特征，最后进行多模态融合和消融评估。实验结果显示，完整三模态方案 F1 为 0.9640，ROC AUC 为 0.9923，说明该流程可以有效支撑内容风险识别；同时行为特征表现很强，我们也在局限性中讨论了数据分布偏差的可能影响。

## 4. 报告摘要可直接使用

> 针对社交网络中垃圾广告、谣言和有害内容识别难以仅依靠文本完成的问题，本文设计并实现了一个多模态内容风控与异常检测流程。项目以公开 EANN-KDD18 Weibo 数据集为基础，将每条样本整理为文本、图片路径、标签、数据划分和行为特征等结构化字段。在模型方法上，文本分支使用 TF-IDF 作为保底基线，并使用冻结 `bert-base-chinese` 提取 768 维文本语义向量；图像分支使用冻结 ResNet18 提取 512 维视觉特征；行为分支对 13 维结构化行为特征进行标准化。随后将不同模态特征按 `sample_id` 对齐并拼接，训练融合分类器。实验在测试集上进行五组消融对比，结果显示完整三模态方案取得 0.9640 的 F1 和 0.9923 的 ROC AUC，说明多模态融合流程具有较好的工程可行性和解释价值。

## 5. PPT 开场稿

> 各位老师好，我们小组的项目是“基于多模态社交网络的内容风控与异常检测”。这个项目关注的问题是，社交平台上的风险内容不一定只体现在文字里，有些信息可能藏在图片中，也可能通过异常发布行为表现出来。因此我们把文本、图片和用户行为三类信息结合起来，构建了一个从公开数据处理、特征提取、多模态融合到实验评估的完整流程。接下来我会从问题背景、数据处理、模型方法、实验结果和项目局限五个方面介绍。

## 6. 最终结论怎么讲

报告和 PPT 推荐把 `text_image_behavior` 作为最终方案，因为它最符合“文本 + 图像 + 行为”的项目主题。

同时要诚实说明：

1. `behavior_only` 指标最高，说明行为特征在当前数据集很强。
2. 这不代表只用行为就一定最好，因为行为字段可能带有数据集分布偏差。
3. `image_only` 较弱，不是失败，而是说明图片信号在当前数据中单独贡献有限。
4. 消融实验的价值就是把这些差异展示出来。
