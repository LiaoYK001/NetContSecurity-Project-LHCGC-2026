# 02 数据字段到模型输入

## 1. 数据从哪里来

默认数据集是 EANN-KDD18 Weibo 多模态谣言检测数据集。开发组没有把完整原始数据提交到 public 仓库，而是在本地把数据转换成项目统一 CSV。

最重要的三张表：

| 表 | 用途 |
| --- | --- |
| `dataset_v1.csv` | 多模态主表，图像、融合和评估都以它为准。 |
| `text_samples.csv` | 文本模型输入表，供 TF-IDF 和 BERT 使用。 |
| `behavior_features.csv` | 行为特征表，供行为分支和融合模型使用。 |

## 2. 为什么要拆成三张表

因为不同模型需要的输入不同：

| 模型分支 | 需要什么 | 对应表 |
| --- | --- | --- |
| 文本分支 | `sample_id`、`text`、`label`、`split` | `text_samples.csv` |
| 图像分支 | `sample_id`、`image_path`、`label`、`split` | `dataset_v1.csv` |
| 行为分支 | `sample_id`、行为数值字段、`label`、`split` | `behavior_features.csv` |
| 融合评估 | 所有分支都要按同一个 `sample_id` 对齐 | `dataset_v1.csv` 作为主表 |

`sample_id` 是整个项目的“对齐钥匙”。文本向量、图像向量、行为向量最后都必须通过它合并。

## 3. 字段逐项解释

| 字段 | 白话解释 | 进入哪个模型 | 报告里怎么写 |
| --- | --- | --- | --- |
| `sample_id` | 每条样本的编号 | 只用于对齐，不作为特征 | 用于保证多模态样本一一对应 |
| `text` | 微博正文清洗后的文本 | TF-IDF、BERT | 表示内容语义信息 |
| `image_path` | 本地图片路径，可为空 | ResNet | 表示图片视觉信息 |
| `label` | 标准答案，`normal` 或 `risk` | 训练和评估 | 二分类风险识别标签 |
| `split` | `train`、`val`、`test` | 所有训练和评估 | 避免训练集和测试集混用 |
| `verified` | 是否认证 | 行为特征 | 账号可信度或身份属性 |
| `reposts` | 转发量 | 行为特征 | 传播强度 |
| `comments` | 评论量 | 行为特征 | 互动强度 |
| `likes` | 点赞量 | 行为特征 | 受关注程度 |
| `engagement_total` | 转发、评论、点赞合计 | 行为特征 | 综合互动量 |
| `interaction_ratio` | 互动量与粉丝数比例 | 行为特征 | 异常互动程度 |
| `followers` | 粉丝数 | 行为特征 | 账号影响力 |
| `following` | 关注数 | 行为特征 | 账号关注结构 |
| `posts_count` | 发帖数 | 行为特征 | 账号活跃程度 |
| `num_image_urls` | 图片数量 | 行为/结构特征 | 内容是否依赖图片 |
| `text_length` | 文本长度 | 行为/结构特征 | 内容复杂程度 |
| `url_mentions` | 链接或图片 URL 数量 | 行为/结构特征 | 外链风险信号 |
| `at_mentions` | @ 用户数量 | 行为/结构特征 | 社交传播方式 |

## 4. 原始数据如何一步步变成模型输入

```text
原始 Weibo 文本文件
-> 读取样本 ID、文本、图片 URL、互动字段
-> 清洗文本，替换用户提及、手机号、URL 等敏感或噪声内容
-> 根据官方 split 文件标记 train / val / test
-> 把 rumor 映射成 risk，把 nonrumor 映射成 normal
-> 生成 dataset_v1.csv、text_samples.csv、behavior_features.csv
-> 文本、图像、行为三个分支分别提取特征
-> 通过 sample_id 合并
```

## 5. 特征维度说明

| 特征 | 维度 | 来自哪里 | 说明 |
| --- | ---: | --- | --- |
| TF-IDF 概率 | 1 | 文本基线模型 | 保底路线，用风险概率代表文本信号 |
| BERT 文本向量 | 768 | `bert-base-chinese` | 标准路线，用上下文语义表示文本 |
| ResNet 图像向量 | 512 | ResNet18 | 把图片转成视觉特征 |
| 行为向量 | 13 | 行为字段标准化 | 表示账号或内容互动特征 |
| 三模态拼接 | 1293 | 768 + 512 + 13 | 最终融合模型输入 |

## 6. D/E 写报告时重点

数据章节不要只列字段，要说明“为什么这些字段有用”：

1. 文本字段用于判断内容语义。
2. 图片路径用于提取视觉特征。
3. 行为字段用于捕捉传播和账号异常。
4. `split` 保证评估公平。
5. `sample_id` 保证多模态对齐。
