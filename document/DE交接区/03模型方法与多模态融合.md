# 03 模型方法与多模态融合

## 1. 总体方法

项目不是只训练一个模型，而是做了三条分支再融合：

```text
文本分支：文本 -> TF-IDF / BERT -> 文本特征
图像分支：图片 -> ResNet18 -> 图像特征
行为分支：行为字段 -> 标准化 -> 行为特征
融合分支：文本特征 + 图像特征 + 行为特征 -> 分类器 -> normal / risk
```

## 2. 文本分支：TF-IDF 和 BERT

### TF-IDF 是什么

TF-IDF 可以理解成“看哪些词更能代表风险内容”。如果某些词在风险样本中经常出现，但在普通样本中不常出现，它们就会得到更高权重。

本项目中 TF-IDF 是保底方案：

```text
text -> TF-IDF 向量 -> 逻辑回归 -> risk_prob
```

优点：快、稳定、容易解释。  
缺点：对上下文理解弱，不太懂复杂语义。

### BERT 在哪里参与

BERT 参与在标准文本特征阶段：

```text
text_samples.csv
-> 分词器处理
-> bert-base-chinese
-> 取 CLS 位置向量
-> 768 维文本向量
-> text_embeddings.csv
```

本项目不微调 BERT 本体，而是冻结 `bert-base-chinese` 作为中文语义特征提取器。这样做更适合短周期课程项目：稳定、可复现、算力要求低。

报告可写：

> 文本分支采用冻结 `bert-base-chinese` 提取 768 维句向量，用于表示每条微博的上下文语义。相比 TF-IDF，BERT 能利用上下文信息，对变体表达和长句含义有更强表示能力。

## 3. 图像分支：ResNet18

图像分支使用 ResNet18，不重新训练图像大模型，只把它当成图片特征提取器：

```text
image_path
-> 读取图片
-> RGB 转换
-> resize 到 224 x 224
-> ImageNet normalize
-> ResNet18
-> 512 维图像向量
```

如果图片缺失、损坏或格式不支持，脚本不删除样本，而是写入 512 维零向量。这样做是为了保证文本、图像、行为三类数据仍然能按 `sample_id` 对齐。

报告可写：

> 图像分支使用冻结 ResNet18 提取 512 维视觉特征，主要用于补充文本难以覆盖的图片风格、截图和视觉线索。对于缺失或无法读取的图片，系统使用零向量保留样本，避免破坏多模态对齐关系。

## 4. 行为分支：结构化特征

行为分支使用 13 维结构化特征，包括认证状态、互动量、粉丝数、关注数、发帖数、图片数量、文本长度等。

处理流程：

```text
behavior_features.csv
-> 选出数值字段
-> 只在 train split 上 fit StandardScaler
-> 转换 train / val / test
-> 13 维行为向量
```

为什么只在 train 上 fit？因为如果用 test 的统计信息参与标准化，就会把测试集信息提前泄露给模型。

## 5. 多模态融合怎么做

标准融合路线：

```text
BERT 文本向量 768 维
+ ResNet 图像向量 512 维
+ 行为向量 13 维
= 1293 维融合特征
-> 标准化
-> 逻辑回归分类器
-> risk_prob
-> pred_label
```

这里使用逻辑回归而不是复杂深度网络，是因为课程项目周期短，逻辑回归更稳定、更容易解释，也便于做消融实验。

## 6. 消融实验是什么

消融实验就是“分别只用某些模态，看看效果变化”。本项目做了五组：

| 模型组 | 使用特征 | 目的 |
| --- | --- | --- |
| `text_only` | BERT 文本向量 | 看文本单独效果 |
| `image_only` | ResNet 图像向量 | 看图片单独效果 |
| `behavior_only` | 行为向量 | 看行为特征单独效果 |
| `text_image` | BERT + ResNet | 看图文融合是否优于纯文本 |
| `text_image_behavior` | BERT + ResNet + 行为 | 看完整三模态方案 |

报告中不要只说“我们做了多模态”，要说“我们通过消融实验验证了不同模态的贡献”。
